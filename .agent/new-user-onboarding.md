# 신규 사용자 온보딩

---

## 목적

처음 앱을 설치한 사용자가 **NotificationListenerService 권한**을 허용하고
첫 번째 자동 거래 내역을 확인하기까지의 경험을 설계한다.

권한 허용율이 핵심 지표 — 권한 없이는 앱의 핵심 기능이 동작하지 않는다.

---

## 플로우

```
앱 최초 실행
    → 스플래시 (로고 + 앱 소개 한 문장)
    → 온보딩 1페이지: "알림으로 자동 기록해요"
    → 온보딩 2페이지: "지원하는 앱 목록" (카카오페이, 토스, 네이버페이 등)
    → 온보딩 3페이지: "권한 허용이 필요해요" + 허용하기 버튼
    → 시스템 알림 접근 권한 설정 화면
    → 권한 허용 완료 → 홈 화면
    → 권한 거부 → 제한 모드 안내 후 홈 화면 (수동 입력만 가능)
```

---

## 화면별 요구사항

### 온보딩 페이지
- 총 3페이지, 스와이프로 이동 가능
- Skip 버튼 제공 (권한 요청은 skip 불가)
- 페이지 인디케이터 표시

### 권한 요청 페이지
- 왜 이 권한이 필요한지 명확히 설명
- "카카오페이, 토스 결제 알림을 읽어 자동으로 가계부에 기록합니다"
- "결제 알림 외의 다른 정보는 수집하지 않습니다" (신뢰 문구 필수)
- 권한 허용 버튼 → 시스템 설정 화면으로 이동
- 나중에 하기 → 제한 모드로 진입

### 권한 거부 시
- 홈 화면 상단에 배너로 권한 요청 유도
- 배너는 X로 닫을 수 있음
- 설정 > 앱 설정에서 언제든 권한 허용 가능

---

## 성공 지표

- 온보딩 완료율 > 80%
- 권한 허용율 > 70%
- 온보딩 후 첫 자동 기록까지 시간 < 5분

---

## 엣지 케이스

| 케이스 | 처리 방법 |
|--------|-----------|
| 앱 재설치 시 | 온보딩 스킵, 권한 상태만 체크 |
| 권한 허용 후 시스템에서 취소 | 설정 화면에서 배너 표시 |
| 권한 설정 화면에서 뒤로가기 | 권한 상태 재확인 후 분기 |
