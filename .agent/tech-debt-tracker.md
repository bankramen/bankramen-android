## 형식

```
### [TD-번호] 제목
- **영역**: 해당 레이어/모듈
- **심각도**: 높음 / 중간 / 낮음
- **설명**: 무엇이 문제인지
- **이유**: 왜 지금 처리하지 않았는지
- **해결 방향**: 어떻게 해결할 것인지
```

## 목록

### [TD-001] NotificationParser 파싱 로직 하드코딩
- **영역**: `service/NotificationParser`
- **심각도**: 중간
- **설명**: 각 앱별 파싱 패턴이 하드코딩된 정규식으로 관리됨
- **이유**: MVP 속도를 위해 일단 하드코딩으로 구현
- **해결 방향**: 서버에서 파싱 패턴을 내려받아 동적으로 업데이트 가능하게 변경

### [TD-002] 카테고리 자동 분류 미구현
- **영역**: `domain/usecase`
- **심각도**: 중간
- **설명**: 거래 내역의 카테고리가 수동 분류만 가능
- **이유**: v1 MVP 범위 밖
- **해결 방향**: 가맹점명 기반 룰 엔진 또는 AI 분류 도입

### [TD-003] 에러 상태 UI 미정의
- **영역**: `presentation/common`
- **심각도**: 낮음
- **설명**: 네트워크 에러, 파싱 실패 등의 에러 상태 UI가 통일되지 않음
- **이유**: 기능 구현 우선
- **해결 방향**: 공통 ErrorScreen Composable 정의 및 적용
